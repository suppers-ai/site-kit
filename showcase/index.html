Composing a runtime from blocks

WAFER is a runtime that composes services from independent processing units called blocks. Each block declares an interface — a small protocol it speaks — and an instance mode controlling how state is shared.

Why blocks?

Blocks are the smallest unit of reuse: a single capability, behind a single interface, with one configuration surface. They cannot reach across the runtime to call each other directly; everything goes through messages.

Concrete example

The HTTP listener block speaks the http-listener interface. It does not know about the database, the router, or any business logic — it only forwards parsed requests to whichever flow it is configured for.

flow main:
  http-listener  port: 8090
  router         routes: [...]
  • One block, one job.
  • State is local; sharing is explicit.
  • Substitution is free as long as the interface matches.
